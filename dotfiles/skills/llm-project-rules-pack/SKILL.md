# LLM Project Rules Pack

Пакет из 4 skill-файлов для полного цикла LLM-проекта.

## Маршрутизация

Определи, что нужно пользователю, и следуй соответствующему файлу:

| Ситуация | Файл |
|----------|------|
| Новый проект, идея, старт с нуля | `llm-project-start.md` |
| Выбор архитектуры: prompt / RAG / tools / workflow / agent / fine-tuning | `llm-architecture-decisions.md` |
| Аудит существующего LLM-проекта | `llm-project-audit.md` |
| Оптимизация production: critic, модель, serving, итерации | `llm-production-hardening.md` |

## Рекомендуемый порядок

1. `llm-project-start.md` — бизнес-постановка и прототип
2. `llm-architecture-decisions.md` — выбор рычагов
3. `llm-project-audit.md` — аудит по 12 правилам
4. `llm-production-hardening.md` — усиление и оптимизация

## Правила

- Не смешивай этапы: сначала бизнес-цель и прототип, потом архитектура, потом аудит, потом production.
- Если пользователь пришёл с конкретным вопросом — направь к нужному файлу, не заставляй проходить все этапы.
- Аудит использует `references/llm-checklist.md` как чеклист.
- Основа методологии: «12 правил разработки проектов с LLM» (vikulin.ai).
